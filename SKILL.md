# 电商视觉文案设计 SOP v2.0

> **核心原则：先定策略，再动笔。必卖理由未确认前，禁止输出任何形式的文案。**
> **核心方法：用买方思维写必卖理由——从买家痛点出发，不堆参数不堆数据。**

## 何时使用

用户提到以下任一关键词时触发：
- 主图文案 / 详情页文案 / 电商文案 / 商品文案 / listing文案
- 转化率优化 / 点击率提升 / 商机分析转文案
- 合规审查 / 广告法 / 平台审核
- 淘宝 / 天猫 / 京东 / 拼多多 / 抖音小店 文案

---

## ⚡ 标准动作清单（每次自动执行，不需要用户提醒）

```
1. 先出必卖理由卡（买方思维） → 等用户确认
2. 每张图必须有背景设计方案（禁止纯白底）
3. 纯文字输出，不用线框/表格/ASCII布局示意
4. 输出前必须自审打分≥80分，不合格自己重写不打扰用户
5. 以上全部自动执行，不等用户提醒
```

## 🏗️ 两阶段工作流（铁律）

```
╔═════════════════════╗         ╔═════════════════════╗
║   第一阶段：策略对齐    ║  ✅确认   ║  第二阶段：文案执行    ║
║  （需要用户决策）      ║ ──────→ ║  （AI 自动完成）       ║
╠═════════════════════╣         ╠═════════════════════╣
║ Step 1 收集信息        ║         ║ Step 3 主图文案        ║
║ Step 2 提炼必卖理由 ⏸️ ║         ║ Step 4 详情页文案       ║
║ ↓ 自审合规后输出给你    ║         ║ Step 5 自审评分(≥80)    ║
║ ↓ 不满意就改          ║         ║ Step 6 制品交付         ║
╚═════════════════════╝         ╚═════════════════════╝
```

### 🚫 铁律（违反 = Skill 失效）

| # | 铁律 | 说明 |
|---|------|------|
| **1** | **必卖理由未确认，禁止写任何文案** | 主图、详情页、FAQ、广告语、短视频脚本 —— 全部禁止 |
| **2** | **必卖理由必须经过合规审查后才能提交给用户** | 不能把违规的理由提交给用户做判断 |
| **3** | **用户确认 = 用户明确说了"可以""通过""没问题"等肯定词** | 默认视为**未确认** |
| **4** | **用户提出修改意见时，只修改必卖理由，不越界到文案** | 理由改到用户满意为止 |
| **5** | **必卖理由必须用买方思维撰写** | ❌ 堆参数堆数据堆技术术语 → ✅ 从买家痛点出发的问句+解决方案 |

### Step 1: 收集信息

向用户索要以下材料，缺项标记清楚：

| # | 必须有 | 用途 |
|---|--------|------|
| 1 | **产品类型判定**：保健食品(蓝帽子) / 化妆品(面膜/护肤品) / 普通食品 / 运动器材 / 其他 | 决定套哪套合规规则 |
| 1.5 | **若为化妆品**：是妆字号还是持有特证？若有特证，具体类别（祛斑美白/防晒）？ | 决定功效宣称范围 |
| 2 | **品牌名 + 公司全称** | 品牌信任模块、法律声明署名 |
| 3 | **SKU列表+价格** | 规格参数表 |

强烈建议有：
| # | 建议有 | 用途 |
|---|--------|------|
| 5 | **商机分析 / 目标人群画像** | ← **必卖理由提炼的核心输入** |
| 6 | 竞品链接/截图 | 差异化定位 |
| 7 | 产品实拍图 | 场景规划更具体 |

> 如果用户提供了商机分析文档或目标人群数据，直接导入使用。

---

### Step 2: 🔑 提炼合规必卖理由卡片（⏸️ 门禁 — 全流程最关键的一步）

这是整个 Skill 的**核心枢纽**。所有后续文案（主图/详情页/FAQ/广告）全部基于这里产出的必卖理由。

#### 2.1 🔴 买方思维铁律（v2.0 新增，不可违反）

**必卖理由必须从"买家的困扰/痛点"出发，禁止堆砌产品参数和技术术语。**

| ❌ 错误写法（堆参数） | ✅ 正确写法（买方思维） |
|---------------------|----------------------|
| "超临界CO₂萃取枸杞籽部位" | "想吃枸杞补一补，但泡煮太麻烦？" |
| "单粒约相当于40粒干枸杞" | "怕糖？不敢随便吃滋补品？" |
| "持有国食健注G202XXXXX号" | "天天觉得累，状态跟不上？" |
| "便携小粒随身携带" | "出差出门，好习惯容易断？" |

**每条必卖理由的标准格式：痛点问题句 + 解决方案句**

让买家看到自己的影子，而不是看到一堆看不懂的数据。

#### 2.2 从输入信息中提取 3-5 条必卖理由

每条必卖理由的结构化卡片格式：

```markdown
## 必卖理由 N：[买方痛点问句]

- **痛点问题句**：谁在什么场景下遇到什么问题？（必须是买家视角的问句或陈述）
- **解决方案句**：产品怎么解决这个问题？（一句话说清楚）
- **信任证据**：专利号/检测报告编号/认证机构+证书号/工艺特点（简短）
- **优先级**：★★★★★ 核心差异 / ★★★★ 重要辅助 / ★★★ 补充
- **适用模块**：主图? 详情? FAQ? 广告?
```

#### 2.3 合规审查（必须逐条过筛）

每条理由必须同时满足：
- [ ] 无绝对化用语（最/唯一/第一/100%/完美…）
- [ ] 无功效暗示（根据产品类型匹配对应合规层）
- [ ] 无医疗化表达（治疗/康复/矫正/治愈…）
- [ ] 证据可查证（专利号/报告编号/认证全称）
- [ ] 利益翻译 ≠ 功效承诺（普通食品尤其严格）
- [ ] **买方思维检查**：这条理由是买家会说的话，还是产品说明书的话？

> 详细合规规则见 `references/compliance-rules.md`，按产品类型自动匹配。

#### 2.4 输出给用户确认（标准格式 — 纯文字）

将审查后的必卖理由整理为以下格式，**仅输出此内容，不附带任何文案初稿**：

```
─── 产品：[产品名] · 合规必卖理由卡 ───
产品类型：[保健食品/普通食品/运动器材/...]
适用合规层：[零层/一层/二层/三层]

#1 [买方痛点问句]
  方案：[解决方案句]
  证据：xxx（编号：xxx）

#2 [买方痛点问句]
  方案：[解决方案句]
  证据：xxx

#3 [买方痛点问句]
  ...

合规检查结论：✅ 已通过[XX]层合规审查，无绝对化用语/无功效暗示/无医疗化表达

─── 请确认以上理由是否准确，或有修改意见 ───
```

#### 2.5 等待用户反馈

| 用户反馈 | AI 响应 |
|----------|---------|
| 「可以」「通过」「没问题」 | ✅ 进入 Step 3 文案阶段 |
| 「理由X改成……」「加上……」 | 修改该条理由 → 重新合规审查 → 再次输出等待确认 |
| 「整体方向不对，重新来」 | 回到 Step 2.1 重新提炼 |
| 其他/沉默/默认 | **保持等待，不主动推进到文案阶段** |

> ⚠️ **再次强调：此步未获明确确认前，禁止执行 Step 3 及之后的所有步骤。这不是建议，是硬性约束。**

### Step 3: 主图5张（仅在 Step 2 确认后执行）

#### 🎨 背景设计铁律（v2.0 新增，每张图都必须遵守）

1. **每张图都必须包含背景设计方案**，❌禁止用纯白底/空白背景（太素、没档次感、像说明书不像商品）
2. 中高端产品（定价¥500+）默认用**暗调质感背景**（深棕偏红#2a1a1a + 顶光阴影层次），让产品跳出来
3. 5张图的背景要有**节奏变化**：
   - 图① 暗调质感首图（高级感第一印象）
   - 图② 暗调+跳色区隔（差异化卖点跳出）
   - 图③ 暗调庄重感（认证/资质类）
   - 图④ 暗调专业冷调（数据/技术类）
   - 图⑤ 实景生活感（最后一张回归场景代入促成下单）
4. 如果产品有品牌色系，背景要与品牌色协调

#### 📝 输出格式铁律（v2.0 新增）

1. **纯文字输出**，❌禁止用ASCII线框、Markdown表格做布局示意（设计师不好复制粘贴）
2. 每张图的固定结构：
   ```
   主标题：xxx
   副标题：xxx
   [辅助语/角标/对比文案/底部小字]（按需）
   背景方案：颜色/光影/纹理描述（设计师能直接理解）
   字体配色：简短说明
   ```

#### 图1 首图·点击率核心

基于已确认的 Top 1-2 条必卖理由，输出最抓眼球的组合。

示例结构：
```
主标题：[核心差异化卖点，≤10字]
副标题：[利益点补充，≤15字]
右下角标1：[信任背书如蓝帽子/认证]
右下角标2：[第二卖点或检测支撑]
背景方案：[暗调质感描述]
字体配色：[主标题色+副标题色+风格说明]
```

#### 图2 痛点共鸣·用户代入

选一条强痛点必卖理由做深度展开。

```
主标题：[买方痛点问句]
副标题：[解决方案句]
对比文案：[旧方式→新方式的简洁对比，≤2行]
底部小字：*[证据编号]
背景方案：[暗调+跳色区域]
字体配色：[对比结论加粗加大]
```

#### 图3 差异化优势·信任背书

资质/认证/批文类必卖理由。

```
主标题：[买方痛点问句]
副标题：[权威背书结论]
角标：[认证标识+可查提示]
背景方案：[暗调庄重/放射光线效果]
字体配色：[白色/金色系，严谨正规]
```

#### 图4 功能/数据支撑

第二功能或数据型必卖理由。

```
主标题：[特定人群痛点问句]
副标题：[批准功能或数据结论]
角标：[认证标识+免责小字]
背景方案：[暗调专业冷调，略带抽象纹理]
字体配色：[白字为主，克制严谨]
```

#### 图5 CTA·场景代入+行动号召

最后一张回归生活场景，促成下单。

```
主标题：[使用场景痛点]
副标题：[便捷性利益点]
信任文案：[多场景适配描述]
背景方案：[实景合成图，暖色调生活光]
字体配色：[深色文字+白色描边，保证可读性]
```

### Step 4: 详情页模块（按需取舍，不必全做）

根据产品需求选择模块组合。每个模块同样遵循：**纯文字 + 买方思维 + 背景方案**。

#### M1 首屏痛点共鸣
- 目标：3秒抓住人，让用户说"这就是我"
- 用已确认的最强痛点必卖理由开篇

#### M2 核心优势展开
- 将**已确认的必卖理由 Top 2-3** 逐个展开：痛点问句→解决方案→证据

#### M3 配方/工艺深度
- 配料表图解 + 工艺流程 + 营养成分数据（如有检测报告）

#### M4 使用场景
- 2-3个典型场景代入（工位/出差/通勤/运动…）

#### M5 品牌/资质信任 ⚠️ 可选
- 专利证书 + 生产许可 + 品牌介绍

#### M6 规格参数+竞品对比
- 参数表格 + 属性差异对照表

#### M7 FAQ
- 从**已确认的必卖理由**反向推演用户顾虑，6道题左右
- 最后一题必须是产品属性界定+免责声明

#### M8 购买引导+法律声明 ⚠️ 可选
- CTA + 免责声明完整版

> **注意**：M5/M8 根据产品需要决定是否做，不是必须。

### Step 5: 自审评分（⏸️ 写完初稿后立即执行，≥80分才允许输出）

| # | 评审项 | 分值 | 扣分标准 |
|---|--------|------|---------|
| 1 | **文案精简度**：无多余文字、无教学式解释 | 20 | 每处多余解释 -2 |
| 2 | **图内可放性**：文案量适合放入图片（≤4行/主图，≤6行/详情）| 20 | 每超1行 -3 |
| 3 | **合规性**：无绝对化用语、无功效暗示、免责声明完整 | 25 | 每处违规 -5 |
| 4 | **结构清晰度**：每张图只有「主副标题+辅助语+背景方案+字体配色」四项 | 15 | 多余板块每个 -3 |
| 5 | **实用性**：设计师拿到就能直接开工，无需再问 | 20 | 缺关键信息每处 -4 |

**总分 < 80？自行重写，不打扰用户直到合格。**

### Step 6: 输出交付

- 以纯文本 Markdown 格式输出完整执行方案
- 作为 artifact 制品交付（`isArtifact=true`）

### Step 7: 归档（项目完成后触发）

输出完成后，执行以下归档操作：

1. **更新 L1 记忆索引**（`references/L1-insight-index.md`）：
   - 新增产品类型条目（如首次遇到该品类）
   - 更新文案策略速查表
   - 更新合规红线速查索引

2. **写入 L4 会话归档**（`references/L4-session-archive.md`）：
   - 按模板填写完整项目记录
   - 记录关键决策、踩坑、可复用经验
   - 下次同类任务先查 L4 再开工

3. **沉淀品类 SOP 模板**（如有新策略模式）：
   - 将本次验证有效的策略写入 L1 必卖理由模式库
   - 标注适用条件和边界

> **目标：每个项目都是下一个项目的加速器。**

## 文案量标准速查

| 位置 | 上限 | 目标 |
|------|------|------|
| 主图标题 | ≤10字 | 5-8字 |
| 主图副标题 | ≤15字 | 8-12字 |
| 单张主图总行数 | ≤4行（v2.0 收紧） | 3行最佳 |
| 详情页单模块 | ≤6行有效文字 | 4-5行 |
| FAQ单条回答 | ≤4行 | 2-3行 |

## 合规规则库（详见 references/compliance-rules.md）

按产品类型自动匹配对应规则层：

| 类型 | 规则层 | 核心红线 |
|------|--------|---------|
| 所有产品 | **零层：通用** | 绝对化用语清零、数据必须有背书、不贬低竞品 |
| 化妆品/面膜/护肤品 | **妆层：广告法+条例** | 禁医疗术语/药妆关联、无功效报告不能宣称功效、特证才可说美白防晒 |
| 蓝帽子保健食品 | **一层：批文铁律** | 只能说批准功能，一字不多不少 |
| 运动器材/体态管理 | **二层：去医疗化** | 禁医学术语、效果必须弱化、禁治疗动词 |
| 普通食品（非保健非药品）| **三层：最严** | **什么功能都不能说**，连暗示都不行 |

详细替换表、禁用词清单、免责声明模板见 `references/compliance-rules.md`
