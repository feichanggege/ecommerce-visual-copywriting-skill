# 电商视觉文案设计 SOP

> 输入：产品信息 + 主图初稿 + 资质边界 + 平台要求
> 输出：合规精简的主图 + 详情页执行方案。每张图只含三项：**画面内容、图内文案、设计说明**。

## 何时使用

用户提到以下任一需求时触发：

- 主图文案、详情页文案、电商文案、商品文案、listing、产品页面
- 淘宝、天猫、京东、拼多多、抖音小店、直播间商品图
- 转化率、点击率、卖点提炼、设计师执行稿
- 合规审查、广告法、平台审核、普通食品、保健食品、运动器材
- “画面 + 文案 + 设计说明”“帮我改成能上图的版本”

不要在这些场景使用：

- 用户只要普通广告长文、品牌故事、社媒贴文，不需要视觉落图结构。
- 用户要求你提供最终法律意见、监管结论或平台审核保证。
- 用户要求编造检测报告、专利号、批准文号、销量、用户评价或功效证明。
- 用户要求自动上架、发布、发给设计师或操作店铺后台。

## 核心原则

1. **先问资质，再写卖点。** 产品类型和证据边界不明时，必须标注缺失，不把猜测写成事实。
2. **先过用户确认门，再写最终稿。** Step 2 的合规版必卖理由必须让用户确认后，才能进入主图和详情页。
3. **只写能落图的内容。** 每张图只保留画面、图内文案、设计说明；删除教程式解释。
4. **普通食品最严。** 普通食品不得暗示任何保健、治疗、改善身体功能或症状关联。
5. **不达 80 分不交付。** 自审评分低于 80 分时，必须自行重写后再输出。

## 核心工作流

```text
1. 收集信息
2. 提炼合规版必卖理由
3. 暂停，等待用户确认
4. 生成主图 5 张
5. 生成详情页模块
6. 合规与可落图自审
7. 输出最终执行方案
```

## Step 1：收集信息

必须确认以下信息。缺项时标注“缺失”，并说明它会影响哪一部分输出。

| 必须项 | 用途 |
|---|---|
| 主图文案初稿或用户口述卖点 | 基础素材 |
| 产品类型：普通食品 / 蓝帽子保健食品 / 运动器材 / 其他 | 匹配合规规则层 |
| 品牌名和公司全称 | 品牌信任模块、声明署名 |
| SKU、规格、价格或价格策略 | 规格参数与 CTA |
| 资质证据：检测报告、批准文号、专利、认证或明确无证据 | 判断哪些卖点可写 |
| 平台：淘宝/天猫/京东/拼多多/抖音小店等 | 平台审核重点 |

建议补充：

- 目标人群画像或商机分析。
- 竞品链接、截图或同类页面。
- 产品实拍图、包装图、详情页旧稿。
- 品牌调性、禁用词、必须出现的免责声明。

## Step 2：提炼合规版必卖理由（必须暂停）

输出 3-5 条必卖理由，每条包含：

```text
目标人群 -> 痛点/场景 -> 产品属性 -> 利益翻译 -> 证据来源 -> 合规风险 -> 优先级 -> 适用模块
```

规则：

- 每条都必须基于用户提供或可验证的事实。
- 没有证据的卖点只能写成“可探索方向”，不能写成事实。
- 必须按 `references/compliance-rules.md` 做合规筛查。
- 输出后必须暂停，等待用户确认或修改。

暂停提示模板：

```text
我先停在必卖理由确认门。请确认这 3-5 条是否能代表真实卖点；确认后我再继续生成主图和详情页执行稿。
```

## Step 3：主图 5 张（用户确认后才能执行）

每张主图只输出三项：画面、图内文案、设计说明。

### 图 1：首图 / 点击率核心

- 画面：产品主视觉 + 背景氛围 + 关键识别点。
- 图内文案：标题、短副标题、1-3 个卖点、必要声明。
- 设计说明：主次层级、点击钩子、免责声明位置。

### 图 2：痛点或场景代入

- 画面：目标用户场景或使用前后的情境对照。
- 图内文案：场景问句或短句，避免功效承诺。
- 设计说明：情绪引导方式，不写医疗化暗示。

### 图 3：差异化优势

- 画面：对比、工艺、原料、规格或数据可视化。
- 图内文案：属性差异 + 证据来源。
- 设计说明：如果没有报告编号，不得写具体数值。

### 图 4：使用场景

- 画面：2-3 个使用场景并列。
- 图内文案：场景标签 + 一句话利益 + 规格。
- 设计说明：生活化表达，不关联疾病或身体功能改善。

### 图 5：购买引导

- 画面：产品全家福 + SKU/价格/CTA 区域。
- 图内文案：规格、组合、适用场景、行动号召。
- 设计说明：CTA 跳色、价格展示方式、声明保留。

## Step 4：详情页模块

按产品需要选择模块，不必全做。每个模块仍然只输出：画面、图内文案、设计说明。

| 模块 | 目标 |
|---|---|
| M1 首屏痛点/场景 | 3 秒让用户知道“这是给我看的” |
| M2 核心优势展开 | 把 Top 2-3 必卖理由展开为视觉模块 |
| M3 配方/工艺/原料 | 展示配料表、工艺流程、证据来源 |
| M4 使用场景 | 2-3 个日常场景或使用方式 |
| M5 品牌/资质信任 | 生产许可、检测、专利、品牌背书 |
| M6 规格参数/对比 | SKU、规格、参数和客观差异 |
| M7 FAQ | 从用户顾虑反推 5-6 个问答 |
| M8 购买引导/法律声明 | CTA、注意事项和免责声明 |

## Step 5：自审评分

输出最终稿前逐项评分。总分低于 80 分时，必须重写。

| 评审项 | 分值 | 扣分标准 |
|---|---:|---|
| 文案精简度 | 20 | 每处多余解释 -2 |
| 图内可放性 | 20 | 主图超过 5 行或详情模块超过 6 行，每超 1 行 -3 |
| 合规性 | 25 | 绝对化、功效暗示、医疗化、无证据数据，每处 -5 |
| 结构清晰度 | 15 | 每张图不是“画面/图内文案/设计说明”三项结构，每处 -3 |
| 实用性 | 20 | 设计师无法直接开工、缺画面层级或声明位置，每处 -4 |

使用 Markdown 输出，结构如下：

```markdown

## 2. 已确认的必卖理由

## 3. 主图 5 张

### 图1：...
- 画面：
- 图内文案：
- 设计说明：

## 4. 详情页模块

## 5. 合规自审评分

## 6. 风险提示与需用户补充的信息
```

## 合规规则库

按产品类型自动匹配 `references/compliance-rules.md`：

| 类型 | 规则层 | 核心红线 |
|---|---|---|
| 所有产品 | 零层：通用 | 绝对化用语清零、数据必须有背书、不贬低竞品 |
| 蓝帽子保健食品 | 一层：批文铁律 | 只能说批准功能，一字不多不少 |
| 运动器材/体态管理 | 二层：去医疗化 | 禁医学术语、禁治疗动词、效果必须弱化 |
| 普通食品 | 三层：最严 | 不写任何功能、保健、治疗或身体改善暗示 |

## 关键失败模式

- 用户没有资质证据，却要求写“改善、调理、修复、治疗、增强”等表达：必须拒绝该表达并给安全替代。
- 用户要直接跳过必卖理由确认门：提醒这是合规门禁，除非用户明确说“按你判断继续”，否则暂停。
- 用户提供竞品截图但没有自身证据：只能做视觉结构和表达参考，不能照搬数据、承诺或认证。
- 用户要求“更猛、更绝对、更有冲击力”：优先提高画面钩子和结构，不提高违法风险。
- 普通食品出现症状/疾病/身体变化关联：整条卖点重写为原料、口感、包装、场景或规格。
